## Abstract

Short-term load forecasting (STLF) is crucial for energy management and integrating renewables, but traditional models often fail to generalize across diverse buildings. While Time Series Foundation Models (TSFM) offer potential, most are transformer-based, making them computationally heavy and unsuitable for edge deployment. We introduce Mix-BEATS , a lightweight and accurate model combining N-BEATS and TSMixer architectures for buildinglevel energy prediction. Adapting TSMixer originated from MLPMixer, originally designed for vision tasks using pure MLP blocks for patchwise and spatial mixing, we leverage its principles for temporal forecasting. Mix-BEATS is trained on hourly smart-meter data from 38,956 buildings and evaluated on a held-out set of 1,000 buildings using the NRMSE metric. We compare Mix-BEATS against state-of-the-art TSFMs such as Lag-Llama, Moirai, Chronos, and Tiny Time Mixers in zero-shot and fine-tuned settings, as well as generic models including Autoformer, N-BEATS, TS-Mixer, etc., under domain-specific training and testing. In both comparisons, Mix-BEATS consistently outperforms the baselines, demonstrating robust generalization and efficiency for real-world STLF tasks. Code: https://github.com/AI-IoT-Lab/Mix-BEATS.

## 1 Introduction

Short-term load forecasting (STLF) is essential for efficient energy management, yet traditional models like ARIMA [15] and basic machine learning approaches struggle to generalize across diverse buildings due to non-linear and heterogeneous consumption patterns. Recently, foundational models tailored for time series forecasting have emerged, including Tiny Time Mixers (TTMs) [8], LagLlama [20], Moirai [22], and Chronos [2]. While these models show strong general forecasting performance, their large transformerbased architectures are often unsuitable for edge deployment and their effectiveness on STLF remains underexplored.

To bridge this gap, we propose Mix-BEATS , a lightweight model that combines the ensemble power of N-BEATS [16] with the efficiency of TSMixer [5]. Pre-trained on hourly energy data from 38,956 real-world buildings, Mix-BEATS is evaluated on a heldout test set of 1,000 commercial and residential buildings using the Normalized Root Mean Squared Error (NRMSE) metric. For comparison, we also benchmarked baseline models such as Autoformer [23], LightTS [3], and TSMixer [5] using domain-specific training.

## Key Contributions

- Mix-BEATS: Alightweight STLF model combining TSMixer and N-BEATS, optimized for efficiency and edge deployment.
- Large-Scale Pretraining: Trained on hourly energy data from 38,956 real-world buildings.
- TSFM Benchmarking: Evaluated against TSFMs (TTMs, Chronos, Lag-Llama, Moirai) on 1,000 held-out buildings in zero-shot and fine-tuned settings with NRMSE metric.
- Generic Model Comparison: Benchmarked against generic models using domain-specific training with MSE metric.

## Algorithm 1: Workflow of the Mix-BEATS Model

```
Input: Input time series x ∈ R 𝑇 , backcast length 𝑇 , forecast horizon 𝐻 , number of stacks 𝑀 , blocks per stack 𝐾 , patch length 𝐿 Output: Final forecast F ∈ R 𝐻 1 Initialize residual & forecast: r 1 , 0 ← x , F ← 0 ; 2 for 𝑚 = 1 to 𝑀 do // Loop over stacks 3 for 𝑘 = 1 to 𝐾 do // Loop over blocks within each stack // TSMixer block processing 4 Reshape: Split r 𝑚,𝑘 -1 to patches as z 1 ∈ R 𝑃 × 𝐿 ; 5 Patch Mixing: z 2 ← LN ( Transpose ( z 1 )) 𝑊 𝑝 𝑚,𝑘 ; 6 z 3 ← z 1 + Transpose ( GELU ( z 2 ) 𝑊 𝑝 ′ 𝑚,𝑘 ) ; 7 Time Mixing: z 4 ← LN ( z 3 ) 𝑊 𝑙 𝑚,𝑘 ; 8 z 5 ← z 3 + GELU ( z 4 ) 𝑊 𝑙 ′ 𝑚,𝑘 ; 9 Reshape: Merge z 5 to actual shape as h 𝑚,𝑘 ∈ R 𝑇 ; // Backcast and Forecast branches 10 𝚯 𝑏 𝑚,𝑘 ← W 𝑏 𝑚,𝑘 · h 𝑚,𝑘 ; 𝚯 𝑓 𝑚,𝑘 ← W 𝑓 𝑚,𝑘 · h 𝑚,𝑘 ; 11 b 𝑚,𝑘 ← Linear ( 𝚯 𝑏 𝑚,𝑘 ) ; f 𝑚,𝑘 ← Linear ( 𝚯 𝑓 𝑚,𝑘 ) ; // Residual update and aggregation 12 r 𝑚,𝑘 ←( r 𝑚,𝑘 -1 -b 𝑚,𝑘 ) ∈ R 𝑇 ; 13 F ←( F + f 𝑚,𝑘 ) ∈ R 𝐻 ; 14 r 𝑚 + 1 , 0 ← r 𝑚,𝑘 ; 15 return F
```

## 2 Methodology

## 2.1 Problem Statement

We address the problem of univariate time series point forecasting in discrete time. Given an observed input sequence as x 1: 𝑇 = { 𝑥 1 , 𝑥 2 , . . . , 𝑥 𝑇 } ∈ R 𝑇 , the goal is to predict ˆ x 𝑇 + 1: 𝑇 + 𝐻 = { ˆ 𝑥 𝑇 + 1 , ˆ 𝑥 𝑇 + 2 , . . . , ˆ 𝑥 𝑇 + 𝐻 } ∈ R 𝐻 . The model is trained using the Huber loss to balance robustness to outliers with smooth optimization (see Appendix D.3). For evaluation, we use NRMSE and MSE metrics, which offer scale-independent accuracy and are well-suited for energy forecasting tasks (see Appendix D.2).

## 2.2 Mix-BEATS

Mix-BEATS is a hybrid neural architecture for univariate time series forecasting (Figure 1), combining the residual learning of N-BEATS with the Patch-Time mixing mechanism of TSMixer.

Input and Output: The model receives a backcast window x ∈ R 𝑇 and aims to predict a forecast window F ∈ R 𝐻 , where 𝑇 and 𝐻 are the input and output lengths, respectively.

Model Architecture: The model consists of 𝑀 stacks, each with 𝐾 blocks. Every block learns to approximate the input and contributes to the overall forecast.

TSMixer Block: Each TSMixer block applies two sequential MLP-based mixing operations: patch mixing and time mixing.

- The residual input r 𝑚,𝑘 -1 is first divided into 𝑃 patches of length 𝐿 as z 1 ∈ R 𝑃 × 𝐿 .
- Patch Mixing: The input is reshaped as Transpose ( z 1 ) ∈ R 𝐿 × 𝑃 and mixed along the patch dimension:

$$z _ { 2 } = L N ( \text {Transpose} ( z _ { 1 } ) ) W _ { m , k } ^ { p } \quad ( 1 )$$

$$z _ { 3 } = z _ { 1 } + \text {Transpose} ( G E L U ( z _ { 2 } ) W _ { m , k } ^ { p ^ { \prime } } ) \quad ( 2 )$$

- Time Mixing: Performed mixing along the temporal dim.

$$z _ { 4 } = L N ( z _ { 3 } ) W _ { m , k } ^ { l } & & ( 3 )$$

$$z _ { 5 } = z _ { 3 } + G E L U ( z _ { 4 } ) W _ { m , k } ^ { l ^ { \prime } }$$

- z 5 is reshaped to recover the time series form h 𝑚,𝑘 ∈ R 𝑇 .

These stages efficiently model inter- and intra-patch dependencies using lightweight MLPs.

Backcast-Forecast Branches: Each TSMixer block produces a hidden state h 𝑚,𝑘 , linearly projected into backcast and forecast components.

$$\Theta _ { m , k } ^ { b } = W _ { m , k } ^ { b } \cdot \mathbf h _ { m , k } \in \mathbb { R } ^ { p } ; \, \Theta _ { m , k } ^ { f } \ = W _ { m , k } ^ { f } \cdot \mathbf h _ { m , k } \in \mathbb { R } ^ { q } \quad ( 5 ) \quad \text {cast}$$

where 𝑝, 𝑞 ≪ 𝑇 , and W 𝑏 𝑚,𝑘 , W 𝑓 𝑚,𝑘 are learnable parameters that capture backward and forward dependencies. Outputs are processed by 𝑔 𝑏 𝑚,𝑘 and 𝑔 𝑓 𝑚,𝑘 to capture trend and seasonality.

$$b _ { m , k } = g _ { m , k } ^ { b } ( \Theta _ { m , k } ^ { b } ) \in \mathbb { R } ^ { T } ; \quad f _ { m , k } \ = g _ { m , k } ^ { f } ( \Theta _ { m , k } ^ { f } ) \in \mathbb { R } ^ { H } \ \ ( 6 )$$

Here, b 𝑚,𝑘 aids residual learning by reconstructing the input, while f 𝑚,𝑘 contributes to the forecast; both use linear functions.

Residual Learning: The model employs residual updates:

$$r _ { m , k } \leftarrow r _ { m , k - 1 } - b _ { m , k } ; \ \ F \leftarrow F + f _ { m , k } \quad \ \ ( 7 ) \quad ^ { 2 } _ { 3 }$$

This process iterates over all stacks and blocks, progressively refining the forecast. The complete workflow of our model is detailed in Algorithm 1.

## 3 Experiments and Results

## 3.1 Datasets

As part of this study, we collected energy datasets from 39,956 real-world buildings-spanning commercial and residential types across multiple countries and years (Appendix A). Brief details of these datasets are provided in Table 2. To avoid bias, we excluded public datasets previously used for pre-training in existing TSFM models in Appendix B. Generic benchmark datasets used for evaluation are summarized in Appendix A.3.

## 3.2 Implementation and Evaluation Setup

All experiments are conducted in JupyterLab using open-source libraries for time series forecasting, including GluonTS 1 , AutoGluon 2 for TSFMs, and TSLib 3 for generic models. Mix-BEATS is pretrained on energy data from 38,956 buildings using an 8-day (192hour) sliding window with a 1-day stride, where each sample includes a 168-hour context to forecast the next 24 hours. The model comprises 3 stacks of 3 TSMixer blocks, with a patch size of 8, hidden dimension of 256, and GELU activation. It is trained for 100 epochs with early stopping on a dual RTX 4090 setup, though GPU use is optional for zero-shot inference. Optimal parameters are chosen through extensive tuning (Figure3). For evaluation, zero-shot forecasting is performed on 1,000 unseen commercial and residential buildings, in the fine-tuned setup, the first 6 months of each unseen building are used for training and the rest for testing, using the past 168 hours to predict the next 24, demonstrating Mix-BEATS' strong generalization.

## 3.5 Ablation Studies

3.5.1 Zero-shot and Fine-tuned Forecasting Results . We evaluated pre-trained models in a zero-shot setting for short-term load forecasting (STLF), assessing their day-ahead forecasting performance across diverse commercial and residential buildings without task-specific fine-tuning. Table 1 compares Mix-BEATS with state-of-the-art TSFMs, including Moirai, Chronos, Lag-Llama, and TTMs.

In the zero-shot setting, Mix-BEATS consistently outperforms other TSFMs on both commercial and residential datasets, demonstrating strong generalization without fine-tuning. For the finetuned setup, we compare Mix-BEATS with fine-tuned Moirai and Lag-Llama. As shown in Table 1, fine-tuning significantly improves performance, especially on residential data. While Mix-BEATS remains competitive on commercial buildings, fine-tuned models like Lag-Llama and Mix-BEATS outperform others in several residential cases. These results highlight the importance of domain adaptation for enhancing foundation model performance in complex, data-rich energy settings. Detailed error analysis is provided in Appendix D.4.

3.5.2 Hyperparameter Sensitivity Analysis and Selection . We conduct a sensitivity analysis to assess the impact of key hyperparameterspatch size (4, 8, 12, 24), sequence length (96, 168, 336, 720), stride (1, 2, 4, 8, 12, 24), and hidden dimension (128, 256, 512, 1024)-on model performance and complexity. As shown in Figure 3, each configuration influences the MSE loss differently. Based on the trade-off between accuracy and compactness, we adopt patch size = 8, sequence length = 168, stride = 24, and hidden dimension = 256 for our final model.

3.5.3 Training and Inference Efficiency Across Models of Varying Sizes: We evaluate the computational efficiency of various models by comparing their training and inference times relative to model size. Mix-BEATS, with only 0.18M parameters, completes training in 34 hours and achieves 0.18 seconds of inference per building. In contrast, larger models like Moirai (14M) and Chronos (46M) take 17 and 13 seconds for inference, respectively. TTMs, with 1M parameters, require 82 hours of training but have a fast inference time of 0.13 seconds. These comparisons highlight the trade-offs between model size, training cost, and inference efficiency, which are critical for real-world deployment in resource-constrained settings.

## 4 Conclusion and Future Work

We propose Mix-BEATS , a hybrid architecture combining NBEATS and TSMixer, for short-term load forecasting (STLF) in smart buildings. The model demonstrated superior performance in both commercial and residential buildings. It outperforms existing models like Chronos and TTMs, achieving the lowest NRMSE in commercial buildings. In residential settings, fine-tuning the TSFM model Moirai led to the best performance, with Mix-BEATS ranking second. Mix-BEATS is also computationally efficient (inference time with 0.18 seconds ) and lightweight ( 0.18M parameters), making it suitable for real-time, edge deployment. However, its performance may degrade in long-term forecasting scenarios due to STLF-specific optimization.

For future work, we aim to scale Mix-BEATS to broader domains by training on diverse datasets and extending its capabilities to classification, long-term forecasting, anomaly detection, and missing data imputation. We also plan to enhance interpretability through block-wise time series decomposition and adapt the model for realtime, multi-resolution forecasting in dynamic environments like smart grids.

## A Dataset Description

This appendix presents a comprehensive overview of the datasets employed for training and evaluating our proposed model.

## A.1 Commercial Building Datasets

A.1.1 EnerNOC. The EnerNOC dataset provides anonymized electricity usage measurements for 100 commercial and industrial facilities across the United States over the full calendar year of 2012. 4

A.1.2 I-Blend. The I-Blend dataset offers minute-level energy consumption data spanning from 2013 to 2017 for an academic institution in Delhi, India. It covers seven buildings: Academic, Lecture, Library, Facilities, Dining, Boys' Dormitory, and Girls' Dormitory, as documented in [19].

## A.2 Residential Building Datasets

A.2.1 CEEW. This dataset includes 3-minute interval electricity consumption data collected from approximately 100 smart meters installed in residential areas of Bareilly and Mathura districts in Uttar Pradesh, India, during the period of May 2019 to October 2021 [1]. The dataset is categorized by district.

A.2.2 Ireland. Collected as part of the StoreNet project [11], this dataset provides 1-minute resolution smart meter data from residential households in Ireland. ## B Details of Time Series Foundation Models (TSFM)

Several recent time series foundation models (TSFMs) offer diverse architectural innovations for large-scale forecasting tasks. Lag-Llama [21] is a decoder-only transformer model tailored for univariate probabilistic forecasting, with 2.45 million parameters and a context length of 1024, trained on over 1 billion observations. Moirai [22], developed by Salesforce AI Research, is an encoderonly architecture with 14 million parameters, pre-trained on the massive LOTSA dataset containing 27 billion time series points across nine domains. It supports a long context window of 5000, allowing broad temporal modeling capabilities. Chronos [2], based on the T5 transformer family, follows an encoder-decoder design with 46 million parameters and a 512-length context, trained on 893K multivariate series for general-purpose sequence modeling. In contrast, N-BEATS [16] is a fully MLP-based model designed for point forecasting, with 2.2 million parameters and an input length of 240, known for its interpretable block-wise architecture. Tiny Time Mixers (TTMs) [8] are lightweight MLP-based models that mix tokens and channels efficiently, using 1 million parameters and a 1536 context window to deliver strong performance with minimal compute. Our proposed Mix-BEATS model is the most lightweight among all, with just 0.18 million parameters and a context length of 168. It integrates the interpretability of N-BEATS with the efficiency of TSMixer layers, enabling accurate point forecasting on the same 282K-series dataset used by N-BEATS and TTMs. Despite its compact size, Mix-BEATS achieves competitive performance and is ideal for edge deployment in real-world smart building applications.

## C Details of Generic Models

In this section, we briefly describe about the generic models used for evaluation with Mix-BEATS. Autoformer [23], a novel decomposition architecture which capacitates progressive decomposition of complex time series. It incorporates with an Auto-Correlation mechanism based on the series periodicity, that outperforms selfattention in both efficiency and accuracy. The main idea of LightTS [4] is to compresses large ensembles into lightweight models while ensuring competitive accuracy. It uses adaptive ensemble distillation that assigns adaptive weights to different base models such that their varying classification capabilities contribute purposefully to the training of the lightweight model. Then identifying Pareto optimal settings w.r.t. model accuracy and model size, thus enabling users with a space budget to select the most accurate lightweight model. Time-Series Mixer (TSMixer) [6], a novel architecture designed by stacking multi-layer perceptrons (MLPs). It is based on mixing operations along both the time and feature dimensions to extract information efficiently. Reformer [12], a memory-efficient transformer for faster training on long-sequences. It employs two techniques to improve the efficiency of Transformers. First, replacing dot-product attention to locality-sensitive hashing, changing its complexity from O( 𝐿 2 ) to O(L 𝑙𝑜𝑔𝐿 ), where L is the length of the sequence. Then, use of reversible residual layers instead of the standard residuals, which allows storing activations only once in the training process instead of N times, where N is the number of layers.

Time-series Dense Encoder (TiDE) [7], a Multi-layer Perceptron (MLP) based encoder-decoder model, for long-term time-series forecasting that enjoys the simplicity and speed of linear models while also being able to handle covariates and non-linear dependencies.
